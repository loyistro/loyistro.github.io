Define Your Hugo Content Structure and Colors

Feb 15, 2025

ⓘ This article belongs to the the My Hugo Theme series.

The Content Structure

Section

  • Definition：Hugo automatically defines content types based on the folder structure.
  • Setup：Each top-level folder, such as content/posts/ or content/projects/, is recognized as a section, with its name derived from the folder name.
  • Reference：https://gohugo.io/methods/page/type/

Type

  • Definition：Manually defined content types, such as posts, projects, or about.
  • Setup：Specified via front matter (type: posts) or determined by the folder structure (e.g., content/posts/ defaults to type “posts”).
  • Reference：https://gohugo.io/methods/page/type/

Layout

  • Definition：Determines which template file is used for rendering the content.
  • Setup：Specified via front matter (layout: single) or follows Hugo’s default behavior.
  • Reference：https://gohugo.io/methods/page/layout/

Taxonomy

  • Definition：A hierarchical or tag-based system to organize content.
  • Setup：Configured via front matter (tags or categories).
  • Reference：https://gohugo.io/content-management/taxonomies/

My Approach

Since my goal is to build a simple personal theme, I have chosen the following approach:

  • Section and Type are quite similar in definition, so I will rely solely on Section for content classification.
  • Layout determines the template file used for rendering content. Hugo’s default behavior is sufficient for my needs, so I won’t manually specify layouts.
  • Taxonomy includes common categories like tags and categories by default. However, I plan to add a custom series taxonomy to organize related articles.

The Color

Although “minimalism” is my goal, I still want to incorporate my own style and preferences. After all, there are already plenty of well-designed Hugo themes out there.

I’ve experimented with many color choices. For example, this website offers a selection of traditional Chinese and Japanese colors, some of which I used in my gofun theme.

However, the final color scheme I settled on was inspired by an Emacs editor theme called Nano Theme. Its creator, Nicolas, introduced me to a color palette called Nord Color, and he also explained his philosophy behind its use in editor themes. I really liked his approach—so much so that he even wrote a paper about it.

Of course, for my personal site, I made some adjustments to his color definitions to make them simpler and more universal:

  • Primary Color: Used for highlighting key information and serving as the site’s main accent.
    • Links
    • Tags
    • Notices
    • Blockquotes
    • Inline Code / Verbatim
    • Footnote Indicators
    • Tables
    • Selected Text
  • Base Colors: Fundamental colors.
    • Foreground
    • Background
  • Secondary Colors: Used for displaying less important information.
    • Summaries
    • Footnote Text
    • Footer
    • Timestamps
    • Dividers

So, I designed my website’s appearance using just four colors, all sourced from the Nord Color palette.

You might have noticed that my website background only features dark-themed colors. That’s because Nord Color was originally designed for dark themes, and switching it to a light theme isn’t as straightforward as it seems. I gave it a try for a while but eventually put it on hold. That said, I’m happy with how it looks now—though I might revisit the idea in the future.
