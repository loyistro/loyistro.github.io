Create a New Hugo Theme Skeleton

Feb 09, 2025   ·   2 min read   ·   #Hugo

ⓘ This post is part of the My Hugo Theme series.

After tweaking two Hugo themes gofun and no-style-please, I still wasn’t satisfied. I realized I wanted something truly my own, designed from the ground up. Maybe it’s a natural next step: once you have a personal blog, the desire for a fully personalized theme starts to feel inevitable.

Luckily, with AI tools, even someone without coding experience (like me) can tackle this challenge. As long as you know how to ask the right questions, you’ll get solid answers—though figuring out what the “right” questions are is a whole other topic.

I’ve named my theme Pace because I like things simple. The name reflects my approach—tweaking the theme at my own pace, adapting it to my needs. It’s going to be a long but exciting journey. If you’re curious about the code, you can check it out here.

Now, let’s dive into the process.

The first step: You might not know exactly what files are needed to design a Hugo theme, but thankfully, Hugo already provides a built-in command to generate a default theme framework:

hugo new theme [name] [flags]

Here, [name] is the name of your theme, and [flags] includes some optional parameters. However, they mainly inherit from the parent parameters and don’t seem particularly useful for theme development (at least, I haven’t found a reason to use them, so I just skipped them).

By default, the generated theme will be placed in the themes folder at the root of your project. Keep in mind that the configuration file is in TOML format. If you, like me, prefer YAML, you can use an online converter like TOML to YAML to switch formats.

Once your theme is ready, you can use the hugo serve command to test it directly inside the theme folder-no need to create a full site:

hugo server --ignoreCache -D --config themes/<your-theme>/hugo.yaml

Make sure hugo.yaml matches your actual config file and format—it could also be hugo.toml.

Now, open your browser and go to http://localhost:1313/, and you’ll see the default theme in action!
