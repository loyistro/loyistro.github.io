工具洁癖

June 25, 2025

我又一次大幅度地修改了这个网站的主题，算是迭代到了 v2.0 版本，而距离上一次的改动可能只过了一个多月，打破了我原本想一年更新一个大版本的想法，不过这不是这次的重点。

在这次修改过程当中，我遇到这么一个问题：我是一个没有编程基础的人，第一次修改主题也是跟着 AI 来开始编写 CSS 文件。这次我想尝试用 Tailwind CSS，可能是之前在某个地方看到它说效率比较高比较新颖吧，自己就觉得有点冲动想换，所以在一些论坛或者主题网站上，找了很多关于用和不用的讨论来看。

于是我就很纠结，一方面是觉得使用新的技术听起来好像很酷，但另一方面也看到了一些用 Tailwind CSS 生成的页面，总觉得会有一种一看就是用「它」做的感觉，好像大家都是通用的一些 UI 模板做出来的一样（大间距大空白圆角的风格），可能很容易联想到所谓的模块化的设计，我并不是很喜欢这样的设计。

所以在纠结了很久之后，自己终于想要尝试一下实际的操作如何，所以在半个小时里面跟着 AI 和官方给出的教程指导一步一步走下来，尝试了大概 15 分钟之后，就已经觉得自己并不需要这样的工具来帮助自己。

虽然这又是一次纠结的结束，但我知道这也映射了我长期以来的一个关于使用工具上的特点：对于同样目的，使用原生的技术跟新兴的技术相比起来，哪一个会比较好？——比如在我学用 Emacs 的时候我就曾经苦恼是用 Vanilla Emacs 还是 Space Emacs。

所有类似问题面临的抉择其实都是一样的：

  • 原生：代表功能全面纯粹，但是不好入门
  • 新兴：集成了很多公认的优秀的插件或者设计，便于新手入门

在多次的纠结体验后我能感受到自己是个有「工具洁癖倾向」的人，我总觉得使用最源头最原始的资料是最可靠的——当然在这种讨论下我会默认投入的时间是值得的，哪怕会比想象中要多。就像 林迪效应 所解释的那样：一个（非易损的）事物已经存在的时间越长，它未来可能继续存在的时间就越久。

我很担心所谓的「新兴技术」在几年之后就变不存在或者被的技术替代，那现在好不容易适应这项技术所花费的时间跟努力在未来就并不在适用了；但原生东西作为基础是一直存在的，无论你后来新发展了多少新的东西，内核还是不变。

我觉得如果学会从这个角度看问题的话，可能很多的纠结就不再必要发生。
